Trip Flow Documentation

1. Trip Request

The trip flow begins when a rider requests a new trip.

API Endpoint

POST /api/trips/rider/request

This endpoint is called by the rider's application to initiate a trip request. It includes details such as pickup and drop-off locations, trip type, and payment method.

Backend Event

TripCreated

When a trip is successfully requested, a TripCreated event is dispatched. This event triggers the driver search process.

2. Driver Search & Assignment

After a trip is requested, the system searches for available drivers.

Driver Search Logic

The driver search logic is handled by the TripService. It identifies nearby drivers who are available and dispatches the trip request to them.

API Endpoint

POST /api/trips/driver/{id}/accept

Drivers accept a trip request by calling this endpoint. The first driver to accept is assigned the trip.

Backend Event

DriverAssigned

Once a driver is assigned to the trip, a DriverAssigned event is dispatched to notify the rider.

3. Driver En Route

The driver is now on the way to the pickup location.

API Endpoint

POST /api/trips/driver/{id}/location

The driver's application periodically sends location updates to this endpoint.

Backend Event

DriverEnRouteToPickup

This event is dispatched when the driver begins moving toward the pickup location, allowing the rider to track the driver's progress.

4. Driver Arrival

The driver has arrived at the pickup location.

API Endpoint

POST /api/trips/driver/{id}/arrive

The driver's application calls this endpoint to notify the system that they have arrived at the pickup point.

Backend Event

DriverArrived

This event is dispatched to notify the rider that their driver has arrived.

5. Trip Start

The rider is in the car, and the trip begins.

API Endpoint

POST /api/trips/driver/{id}/start

The driver's application calls this endpoint to start the trip.

Backend Event

TripStarted

This event is dispatched to signal that the trip is officially underway.

6. Location Completion (for Multi-Stop Trips)

For trips with multiple stops, this phase marks the completion of an intermediate stop.

API Endpoint

POST /api/trips/driver/{id}/complete-location

The driver's application calls this endpoint when they arrive at one of the intermediate locations.

Backend Event

TripLocationCompleted

This event is dispatched to notify the system that a stop has been completed, and the trip can proceed to the next destination.

7. Trip Completion

The trip has reached its final destination.

API Endpoint

POST /api/trips/driver/{id}/complete

The driver's application calls this endpoint to mark the trip as complete.

Backend Event

TripCompleted

This event is dispatched to finalize the trip, calculate the fare, and process payment.

8. Trip Cancellation

A trip can be canceled by either the rider or the driver at different stages of the trip.

Rider Cancellation

POST /api/trips/rider/cancel

A rider can cancel the trip before it starts. Depending on the timing, a cancellation fee may apply.

Driver Cancellation

POST /api/trips/driver/cancel

A driver can cancel the trip, for instance, if the rider does not show up at the pickup location.

Backend Events

Corresponding events are dispatched for rider and driver cancellations to handle the state change and any applicable fees.
